# Функциональные требования

## Назначение сервиса

Task Hub хранит задачи и предоставляет REST API для работы с ними.

Основная сущность системы — задача. Задача имеет заголовок, статус, приоритет и необязательный срок исполнения.

## Модель задачи

Поля задачи:

| Поле | Тип | Описание |
| --- | --- | --- |
| id | uuid | Генерируется сервером. Не изменяется после создания. |
| title | string | Заголовок задачи. Обязательное поле. |
| status | enum | Статус задачи: `todo`, `in_progress`, `done`. По умолчанию `todo`. |
| priority | enum | Приоритет задачи: `low`, `medium`, `high`. |
| due_date | timestamp with timezone | Срок исполнения. Необязательное поле. |
| created_at | timestamp with timezone | Техническая дата создания записи. |
| updated_at | timestamp with timezone | Техническая дата последнего обновления записи. |
| deleted_at | timestamp with timezone | Техническое поле soft-delete удаления. |

## Создание задачи

Эндпоинт:

```http
POST /tasks
```

Правила:

- `title` обязателен и не может быть пустой строкой.
- `priority` обязателен.
- `status` можно не передавать, в этом случае используется `todo`.
- `due_date` можно не передавать.
- Если `due_date` передан, он должен быть timestamp с timezone.
- При создании `due_date` не может быть в прошлом.
- `id` генерируется сервером.
- Клиент не может управлять техническими полями `id`, `created_at`, `updated_at`, `deleted_at`.

После успешного создания API возвращает `201 Created` и созданную задачу, включая `id`.

## Получение задачи

Эндпоинт:

```http
GET /tasks/:id
```

Правила:

- Возвращается только неудалённая задача.
- Если задача не найдена, возвращается `404`.
- Если задача была soft-delete удалена, она также считается недоступной и возвращается `404`.

## Получение списка задач

Эндпоинт:

```http
GET /tasks
```

Поддерживаются одновременно:

- пагинация через `offset` и `limit`;
- фильтр по `status`;
- сортировка по `due_date` или `priority`;
- направление сортировки `asc` или `desc`.

Удалённые задачи не возвращаются в списке.

## Обновление задачи

Эндпоинты:

```http
PATCH /tasks/:id
PUT /tasks/:id
```

`PATCH` выполняет частичное обновление. В теле запроса должно быть передано хотя бы одно изменяемое поле.

`PUT` выполняет полную замену изменяемой части задачи. Для `PUT` обязательны ключи:

- `title`;
- `status`;
- `priority`;
- `due_date`.

`due_date` при `PUT` может быть `null`.

При обновлении разрешено установить `due_date` в прошлое. Ограничение на дату в прошлом действует только при создании задачи.

`id` задачи изменять нельзя.

## Удаление задачи

Эндпоинт:

```http
DELETE /tasks/:id
```

Удаление выполняется как soft-delete через заполнение поля `deleted_at`.

Физически запись из базы данных не удаляется.

После удаления задача недоступна через публичные эндпоинты.

## Коды ошибок

| Код | Ситуация |
| --- | --- |
| 400 | Некорректный формат данных, неверный enum, неверный UUID, пустой PATCH, попытка изменить id. |
| 404 | Задача не найдена или была удалена. |
| 422 | Данные корректны по формату, но нарушают бизнес-правило. Например, `due_date` в прошлом при создании. |
| 500 | Неожиданная ошибка приложения. |
